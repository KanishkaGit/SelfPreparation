Object Oriented Programming - Introduction

Overview

Object Oriented Programming can be defined as a paradign or design of solving software programming problems using the concept of collaborating objects. Object collaorate by sending messages to each other. Hence OOP focuses more on the actual object that is being operated on and the data inside it rather than the actions and logic behind it.
Hence OOP relies on the data and behaviors of the object to define the operations and logics and hence is more geenralized over the entirity of the solution.

OOP relies on two major concepts for implementation of the same. They would be as follows.

  1. Objects
  2. Classes

Objects

Objects can be defined as real life entities that have some data and behaviors associated with it. Examples of real life entites would be Cars, House, Account, Employee etc. In OOP, an object is defined as a set of two group of it's characteristics -

  1. States - These are the characteristics of an Object which define it in a way and describe it in general.
  2. Behaviors - These are specific distictive actions thatn can be done by the object.
Put another way, an object encapsulates data and the functions that operate on that data. The data is usually hidden from other objects so that the only way to affect the data is through the objectâ€™s functions (or methods)
A simple example of an object would be a person. Logically, you would expect a person to have a name. This would be considered a property of the person. You could also expect a person to be able to do something, such as walking or driving. This would be considered a method of the person.

If we consider our Dog as an object, inorder to differentiate from(or characterize like) any other Dog, it sould have some State - name, color, breed and will also have some behaviors - barking, fetching, eating.

In programming, the states are represented using variables and behaviors using methods. So, in programming terms, an Object can also be defined as a software bundle of variables and related methods.

Classes

If we have understood objects, we already know that an object is an entity with a set of properties and behaviors. It should be quite evident that aan object can have the some properties and behaviors that are common with other objects. For example, your dog and my dog, both will have a name, both will bark and run and eat at more. Which is where classes come into play.
A class defines a blueprint or template of what the object would consist of. Hence a class will have some properties and behaviors, which will be the properties and behaviors of the object that is being created from the class. This creation of object from a class is called instatiation of an Object. Hence in order to get an entity from a blueprint, we create an instance of the class (blueprint) called an object(entity).

  • A class is a description (model) of real objects and events called entities.
  • A class consists of characteristics behaviors that are characteristics of the object that is instantiated from it.

Object oriented Programming paradigm

The OOP paradigm consists of working with Classes and Objects and it's Implementations, and use of the fundamental principles and concepts of OOP : Inheritance, absraction, encapsulation and Polymorphism.
  • Inheritance
    Inheritance is a fundamental principle of OOP which allows one class to inherit or use properties and behaviors of another ,more geenral class. For example a Lion is an animal which has similar characteristics of a Cat like it will have four legs, are predators and hunt their prey. Hence Lion will a class that could inherit property from a Cat. In this way, the characteristics can be defined once and can be used for all the animals that are cats in some way or other can be grouped into one.
    This also helps in the fact that if one day, all cats decide to buy their food and not hunt, it can be done in one class and the same will be reflected to the entire family. This helps easy maitanibility.
  • Abstraction
    Abstraction is another core principle of OOP, that means to work with something where we only know How to use and not concerned with what it is doing internally to accomplish that. A very convinient example would be of a Television, where we know how to operate the remote, go to our favourite channes, and watch what we want to, without actually knowing about how it is acheiving the same.
  • Encapsulation
    Encapsulation is another key principle of OOP, where the object provides the user with the essential information and hide all the internal details, if not needed toAn example of encapsulation would be a secratary using a laptop, who knows only the screen and not the internal details of the laptop because she doesnt need to. She only has access to apps which is required for her day to day operations.
    A basic doubt in people's mind is about how is abstraction different from encapsulation. Please refer below details.
    • Abstraction has to do with separating interfaces from implementation. (We don't care how it is doen, if it does what it is needed to).
      Encapsulation has to do with access of data, on what is supposed to be exposed and what not.
  • Polymorphism
    The Next important concept of OOP is polymorphism. Polymorphism allows treating objects of derived class as objects of its base class. For example, all objects of Cats like cheetah, Lion, House Cat will have different ways to catch their prey, but using polymorphism, we can make the catching prey behavior of each derived class to act differently by using the same name but changing the behavior in the derived class. This is also called as overriding.
